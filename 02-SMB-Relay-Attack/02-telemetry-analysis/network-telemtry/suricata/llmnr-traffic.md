# Suricata Telemetry – LLMNR Traffic

During the attack simulation, Suricata observed Link-Local Multicast Name Resolution (LLMNR) traffic generated by the victim host when DNS resolution failed.

LLMNR uses UDP port 5355 and multicast address 224.0.0.252.

## Example Event

Timestamp: 2026-03-08 14:22:31  
Source IP: 192.168.4.16  
Destination IP: 224.0.0.252  
Protocol: UDP  
Destination Port: 5355  

Event Type: Network Protocol Detection  

## Interpretation

The Windows client attempted to resolve a hostname using LLMNR after DNS resolution failed.

Because LLMNR does not authenticate responses, an attacker on the same network can respond to this query and impersonate the requested host.

This behavior allows tools such as Responder to capture NTLM authentication attempts from the victim.
