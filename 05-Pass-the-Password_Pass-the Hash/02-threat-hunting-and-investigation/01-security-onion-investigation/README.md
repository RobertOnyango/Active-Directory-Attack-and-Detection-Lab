# Security Onion Alerts and Investigations

## Security Onion Alerts

Due to the attack activities performed during the simulation, alerts were generated across the SOC monitoring environment. Security Onion IDS produced four alerts associated with the attack chain. We previously investigated the high-severity alert, **"Possible Kali Linux hostname in DHCP Request Packet,"** during the IPv6 Man-in-the-Middle Attacks lab. In this walkthrough, we will revisit the alert and associated telemetry to determine whether it represents a true positive security incident and identify defensive actions that can be taken to mitigate similar attacks.

## Security Onion Investigations

### Identify the Alert Source

The endpoint that triggered the alert was identified as:

```text
IP Address: 192.168.4.11
```

This system corresponds to the attacker's Kali Linux host used throughout the lab.

![alt text](<Figure 16.png>)

---

### Identify Additional Alerts Associated with the Host

The first step in the investigation is to determine whether additional alerts were generated by or associated with the source IP address.

Investigation findings revealed that the alert under investigation was the only alert directly associated with the host during the selected timeframe.

### Findings

- Source IP Address: `192.168.4.11`
- Alert Count: 1
- Alert Name: `Possible Kali Linux hostname in DHCP Request Packet`

This confirms that Security Onion successfully identified the presence of the attacker-controlled host on the network.

---

### Investigate Network Activity

Next, investigate the network activity associated with the source IP address using the Security Onion Dashboards.

Reviewing the dashboard revealed a summary of the destination ports communicating with the host, either as a source or destination endpoint.

### Findings

- HTTPS traffic over TCP port `443`
- DNS traffic over UDP/TCP port `53`
- DHCP traffic over UDP port `67`

---

### Analyze Observed Network Traffic

The network traffic observed during the investigation included:

#### HTTPS Traffic (Port 443)

Normal HTTPS web traffic generated through the Mozilla Firefox browser.

This activity is expected and does not immediately indicate malicious behavior.

#### DNS Traffic (Port 53)

DNS requests were sent to the network gateway:

```text
192.168.4.1
```

This is expected behavior as systems require DNS services to resolve hostnames and locate resources within the environment.

#### DHCP Traffic (Port 67)

DHCP traffic was observed between the rogue host and the network gateway.

This traffic is particularly important because it is likely responsible for triggering the Security Onion alert.

---

### Analyze DHCP Activity

![alt text](<Figure 19.png>)

Reviewing the DHCP logs revealed the host at `192.168.4.11` sending DHCP packets to the gateway on destination port `67`.

This indicates that the host was attempting to obtain or renew network configuration information through the DHCP service.

The observed traffic is consistent with:

- DHCP Discover
- DHCP Request

messages commonly used during the DHCP lease process.

### Findings

- Source IP: `192.168.4.11`
- Destination IP: `192.168.4.1`
- Destination Port: `67`
- Protocol: DHCP

This confirms that the rogue device requested access to network resources and successfully joined the network.

---

## Investigation Conclusion

At this stage of the investigation, we have identified a rogue host operating from IP address `192.168.4.11` that is suspected to be running Kali Linux, a distribution commonly used for penetration testing and offensive security operations. Security Onion successfully detected the device through DHCP-related network activity and generated an alert that provided the starting point for the investigation.

While the observed DHCP, DNS, and HTTPS traffic is not inherently malicious, the presence of an unauthorized Kali Linux host on the network warrants further investigation. The next step is to pivot to endpoint telemetry and authentication logs to determine what activities the host performed after gaining access to the Active Directory environment.
