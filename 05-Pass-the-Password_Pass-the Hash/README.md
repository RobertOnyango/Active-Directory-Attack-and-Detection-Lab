## 📌 Overview

Pass the Password and Pass the Hash are credential abuse techniques that allow attackers to move laterally throughout an Active Directory environment after obtaining valid authentication material.

In a Pass the Password attack, the attacker authenticates using stolen plaintext credentials. In a Pass the Hash attack, the attacker authenticates using NTLM password hashes without knowledge of the user's actual password.

This lab demonstrates how compromised credentials can be reused to access multiple systems, attempt credential extraction, and facilitate lateral movement across an enterprise network.

The attack begins with credentials obtained during previous attack phases in this series. The attacker then uses SMB-based tooling to authenticate to multiple systems, enumerate accessible hosts, and attempt credential dumping operations.

The attack was simulated in an Active Directory environment and monitored with:

- **Security Onion (Suricata IDS)**
- **Splunk (Windows Event Logs)**
- **Wireshark (packet analysis)**

The objective of the lab is to demonstrate:

- How Pass the Password attacks are performed
- How Pass the Hash attacks are performed
- The telemetry generated during credential abuse activity
- How analysts investigate NTLM-authenticated lateral movement
- Detection opportunities for SOC environments
- Visibility limitations in network telemetry

## ⚔️ Attack Simulation

The attack was performed from a Kali Linux attacker machine using compromised domain credentials.

### Steps

1. Attacker validates compromised credentials against systems within the domain using CrackMapExec.
2. Attacker identifies hosts that accept the compromised credentials.
3. Attacker attempts to dump local account hashes using SMB-based administrative access.
4. Attacker attempts remote command execution using PsExec.
5. Attacker extracts credential material using SecretsDump.
6. Attacker performs offline password cracking using Hashcat.
7. Attacker attempts Pass the Hash authentication using recovered NTLM hashes.
8. Authentication activity generates endpoint and network telemetry across the environment.

## 📡 Telemetry Artifacts

The attack generated observable artifacts across both endpoint and network telemetry.

Examples include:

- NTLM-authenticated network logons (Event ID 4624 – Logon Type 3).
- Failed and successful authentication activity.
- Credential Manager access events (Event ID 5379).
- Security Account Manager (SAM) object access requests (Event ID 4656).
- Access requests performed by `lsass.exe`.
- SMB connection attempts targeting TCP port 445.
- Multiple host authentications originating from a single source IP.
- Repeated TCP SYN packets and retransmissions observed in packet captures.

Artifacts were analyzed using:

- Security Onion alert triage.
- Splunk SIEM investigations.
- Windows Security Event Logs.
- Wireshark packet captures.

Screenshots and analysis are available in the **telemetry-analysis/** directory.

## 🛡️ Enterprise Mitigations

Recommended mitigations include:

- Disable NTLM authentication where operationally feasible.
- Prefer Kerberos authentication within Active Directory environments.
- Deploy Microsoft LAPS or Windows LAPS.
- Prevent local administrator password reuse.
- Restrict SMB administrative access.
- Segment administrative and user networks.
- Monitor NTLM authentication activity.
- Audit Security Account Manager (SAM) access requests.
- Implement Privileged Access Management (PAM).
- Apply least privilege principles to administrative accounts.

## 🧑‍💻 Why This Matters

Credential abuse remains one of the most common techniques used by attackers after initial compromise.

Unlike malware-based attacks, Pass the Password and Pass the Hash frequently rely on legitimate authentication mechanisms, making them difficult to distinguish from normal administrative activity.

As a result, detecting these attacks requires:

- Authentication monitoring
- Telemetry correlation
- Behavioral analysis
- Visibility across both endpoint and network data sources

rather than relying solely on traditional signature-based detections.

---

## 💻 Lab Takeaways

This lab demonstrates several real-world detection challenges:

- Credential abuse often appears as legitimate user activity.
- NTLM-only authentication can provide strong evidence of lateral movement.
- Host telemetry provided significantly greater visibility than packet captures during the investigation.
- Packet captures alone were insufficient to confirm credential dumping activity.
- Correlating authentication events, object access telemetry, IDS alerts, and packet captures was essential to reconstructing the attack chain.
- Sensitive object access requests can provide valuable investigative context during credential abuse incidents.
